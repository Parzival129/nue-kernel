  ---
  Codebase Overview                                                                                                                                                    
          
  This is a bare-metal i386 (x86 32-bit) hobby kernel that boots via the GRUB Multiboot standard. It's at an early but solid foundational stage.                       
                                                                                                                                                                     
  ---                                                                                                                                                                  
  Boot Sequence
                                                                                                                                                                       
  kernel/arch/i386/boot.S — Entry Point                                                                                                                                
                                                                                                                                                                       
  The very first code that runs after GRUB hands off control:
  1. Embeds a Multiboot header (magic 0x1BADB002, flags for page-alignment and memory map) in .multiboot so GRUB recognizes it.
  2. Declares a 16 KiB stack in .bss.
  3. _start sets %esp to the top of that stack, calls _init (global constructors), then calls kernel_main.
  4. If kernel_main ever returns: cli + infinite hlt loop — safe halt.

  kernel/arch/i386/crti.S / crtn.S — C Runtime Init

  These provide the prologue and epilogue of the _init/_fini functions. GCC's crtbegin.o/crtend.o fills in the body. Together they implement the standard C++ global
  constructor/destructor mechanism.

  kernel/arch/i386/linker.ld — Memory Layout

  - Kernel loads at 1 MiB (conventional for protected mode kernels).
  - Sections: .text → .rodata → .data → .bss, each page-aligned (4 KiB blocks).
  - .multiboot is placed at the very start of .text so GRUB finds it.

  ---
  Kernel Main (kernel/kernel/kernel.c)

  kernel_main is the C entry point. It runs three initializers in sequence then loops forever halting:

  terminal_initialize()  → sets up VGA text output
  gdt_install()          → loads the Global Descriptor Table
  idt_install()          → loads the Interrupt Descriptor Table + PIC

  Then prints the "nue kernel" ASCII art banner and enters hlt loop, which keeps the CPU idle while the IDT handles keyboard IRQs.

  ---
  Terminal / VGA Driver (kernel/arch/i386/tty.c, vga.h)

  Implements a VGA text mode terminal (80×25, memory-mapped at 0xB8000):

  - vga.h: Color enum + inline helpers to pack a character+color into a uint16_t VGA cell.
  - terminal_initialize(): Clears the screen, sets light grey on black.
  - terminal_putchar(): Handles \n, \b, \t, regular chars. Wraps columns and calls scrollup() when the screen fills.
  - scrollup(): Shifts all 25 rows up by one, clearing the bottom row.
  - update_cursor(): Writes the hardware VGA cursor position via I/O ports 0x3D4/0x3D5.
  - PS/2 keyboard → ASCII: ascii_map[] and ascii_shift_map[] translate Scan Code Set 1 scancodes. ps2_to_ascii() handles shift state (make/break codes).

  ---
  Global Descriptor Table (kernel/arch/i386/gdt.c, gdt.h, gdt_init.s)

  Sets up the CPU's memory segmentation (required for protected mode):

  ┌───────┬──────────┬─────────────────┬────────┐
  │ Index │ Selector │     Segment     │ Access │
  ├───────┼──────────┼─────────────────┼────────┤
  │ 0     │ 0x00     │ Null descriptor │ —      │
  ├───────┼──────────┼─────────────────┼────────┤
  │ 1     │ 0x08     │ Kernel Code     │ 0x9A   │
  ├───────┼──────────┼─────────────────┼────────┤
  │ 2     │ 0x10     │ Kernel Data     │ 0x92   │
  ├───────┼──────────┼─────────────────┼────────┤
  │ 3     │ 0x18     │ User Code       │ 0xFA   │
  ├───────┼──────────┼─────────────────┼────────┤
  │ 4     │ 0x20     │ User Data       │ 0xF2   │
  └───────┴──────────┴─────────────────┴────────┘

  All segments are flat (base=0, limit=4 GiB), using ring 0 for kernel and ring 3 for user (future use). gdt_init.s uses the lgdt instruction to load it and does a far
   jump (ljmp $0x08) to flush the segment cache.

  ---
  Interrupt Descriptor Table (kernel/arch/i386/idt.c, idt.h, idt_init.s, isr.s)

  The most complex subsystem. It handles all CPU exceptions and hardware interrupts:

  isr.s — Assembly Stubs

  Three GAS macros generate stub functions:
  - ISR_NOERRCODE num — pushes dummy 0 error code + interrupt number, jumps to common handler.
  - ISR_ERRCODE num — CPU already pushed error code; pushes interrupt number, jumps to common handler.
  - IRQ_HANDLER num offset — same pattern for hardware IRQs (mapped to IDT 32–47).

  isr_common_handler is the shared trampoline:
  1. pusha — saves all GP registers.
  2. Saves DS, switches all segment registers to kernel data segment (0x10).
  3. push %esp — passes pointer to the saved register state as struct interrupt_frame*.
  4. Calls C function isr_handler.
  5. Restores all registers, cleans up stack, iret.

  idt.c — C Setup and Dispatch

  - outb/inb: Direct I/O port access via inline assembly.
  - io_wait(): Writes to port 0x80 to pad timing for slow hardware.
  - pic_remap(): Remaps the 8259 PIC from its default conflicting range (0x08–0x0F, which clashes with CPU exceptions) to 0x20–0x2F. This is mandatory in protected
  mode.
  - pic_send_eoi(): Sends End-Of-Interrupt signal to PIC so it can fire again.
  - idt_install(): Registers all 256 entries (32 exceptions + 16 IRQs), remaps PIC, masks everything except IRQ0 (timer) and IRQ1 (keyboard) via outb(0x21, 0xFC), then
   sti to enable interrupts.
  - isr_handler(): The C dispatcher — for exceptions it prints the name and halts; for IRQ1 (keyboard) it reads port 0x60, converts scancode to ASCII via ps2_to_ascii,
   and calls putchar.

  ---
  LibC (libc/)

  A minimal freestanding libc for the kernel:
  - string/ — memcpy, memset, memmove, memcmp, strlen
  - stdlib/ — abort
  - stdio/ — putchar (calls terminal_putchar), puts (calls printf)

  The kernel links against this via -lk (the kernel's own libc copy).

  Current State Summary

  Working:
  - Multiboot boot via GRUB
  - VGA text output with scrolling, cursor, backspace, tab
  - GDT (flat segmentation, kernel + user segments defined)
  - IDT with all 32 CPU exceptions + 16 hardware IRQs
  - PIC remapping
  - PS/2 keyboard input → character display (IRQ1)
  - Division-by-zero exception handler (commented-out test exists)

  Not yet implemented:
  - Command parsing (the > prompt exists but typed input isn't processed as commands)
  - User space / ring 3 switching
  - Paging / virtual memory
  - Memory allocator (malloc/free)
  - Filesystem
  - Processes / scheduling
  - printf with full format support (likely uses a basic hosted version; a freestanding printf is needed)

  ---
  Recommended Next Steps (in order)

  1. Freestanding printf — The kernel uses ar 16printf but links -lk. You likely need a proper kernel printf that calls terminal_putchar directly, handling %s, %d, %u, %x
  at minimum.
  2. Command-line shell — Buffer keyboard input in isr_handler (IRQ1) into a line buffer, echo it, and on \n dispatch to a command parser. This gives you an
  interactive kernel shell.
  3. Physical Memory Manager (PMM) — Use the Multiboot memory map (passed in %ebx at boot, currently ignored) to build a bitmap or stack-based page frame allocator.
  4. Paging — Set up page directory/tables, enable CR0.PG. This is prerequisite to a heap and user space.
  5. Kernel heap (kmalloc) — A simple first-fit allocator over a known kernel region.
  6. Processes & Scheduling — TSS (Task State Segment, currently no segment 5 in your GDT), context switching, a basic round-robin scheduler.
  7. User space — Ring 3 segments are already in the GDT (0x18/0x20). You need iret to ring 3 + system call interface.
